2. Coursework Tasks (Step-by-Step)
Part 1: Service Architecture & Setup (10 Marks)
1. Project & Application Configuration (5 Marks):
-BootstrapaMavenprojectintegratingaJAX-RSimplementation(e.g.,Jersey)andalightweight
servlet container or embedded server.
-Implementasubclassof javax.ws.rs.core.Application andusethe @ApplicationPath("/api/v1")
annotation to establish your API’s versioned entry point.
-Question: When returning a Java object from a method, it is automatically serialised into
JSON.Explaintheroleofa MessageBodyWriter oraJSONprovider(likeJackson)inthiscon-
version process.
2. The ”Discovery” Endpoint (5 Marks):
-Implement a root ”Discovery” endpoint at GET /api/v1 . This should return a JSON object
providing essential API metadata: versioning info, administrative contact details, and a
map of primary resource collections
(e.g., ”workspaces” - >”/api/v1/workspaces”).
-Question: REST architecture dictates that APIs should be strictly ’stateless’. Define what
statelessness means in this context and explain why it makes cloud APIs easier to scale
horizontally across multiple servers.
Part 2: Workspace Management (20 Marks)
1. Workspace Resource Implementation (10 Marks):
-Develop a WorkspaceResource class to manage the /api/v1/workspaces path.
-GET /: Provide a comprehensive list of all ML workspaces.
-POST /: Enable the creation of new workspaces. Ensure the service returns appropriate
feedback upon success.
-GET /{workspaceId} : Allow users to fetch detailed metadata for a specific workspace.
-Question: DiscusshowimplementingHTTP Cache-Control headersonthe GETworkspaces
endpoint could improve performance for the client and reduce unnecessary processing
load on the server.
2. Workspace Deletion & Safety Logic (10 Marks):
-Implement DELETE /{workspaceId} to allow workspace removal.
-Business Logic Constraint: To prevent data orphans, a workspace cannot be deleted if it
still has models assigned to it. If a deletion is attempted on a populated workspace, your
service must block the request and return a custom error response (as detailed in Part 5).
-Question: Ifaclientneedstoverifywhetheraspecificworkspaceexistsbutwantstosave
bandwidthbynotdownloadingtheentireJSONbody,whichHTTPmethodshouldtheyuse
instead of GET? Explain your reasoning.

Part 4: Deep Nesting with Sub-Resources (20 Marks)
A key requirement is to maintain a historical log of evaluation metrics for every model to track
concept drift and performance over time.
1. The Sub-Resource Locator Pattern (10 Marks):
-In yourModelResource class, implement a sub-resource locator method for the path
{modelId}/metrics . This method should return an instance of a dedicated
EvaluationMetricResource class.
-Question: You can place annotations like @Produces(MediaType.APPLICATION_JSON) at ei-
ther the class level or the individual method level. What is the benefit of class-level place-
ment, and how does method-level overriding work?
2. Historical Data Management (10 Marks):
-WithinEvaluationMetricResource ,implement GET /tofetchhistoryand POST /toappend
new evaluation metrics for that specific model context.
-Side Effect: A successful POSTto a metric must trigger an update to the latestAccuracy
fieldonthecorrespondingparent MachineLearningModel objecttoensuredataconsistency
across the API.
Part 5: Advanced Error Handling, Exception Mapping & Logging (30 Marks)
YoumustensuretheAPIis”leak-proof”—itshouldneverreturnarawJavastacktraceoradefault
server error page. Furthermore, the API requires observability through proper request and
response logging. Implement specific Exception Mappers and Filters for these scenarios.
1. Resource Conflict (409) (5 Marks):
-Scenario: Attempting to delete an ML Workspace that still has Models assigned to it.

2. Dependency Validation (422 or 400) (10 Marks):
-Scenario: A client attempts to POSTa new Model with a workspaceId that does not exist.
-Task:Create a LinkedWorkspaceNotFoundException . Use an Exception Mapper to return

-Question: HTTPstatuscodesarecategorisedintoclasses(e.g.,2xx,4xx,5xx). Explainfun-

3. State Constraint (403 Forbidden) (5 Marks):
-Scenario: A model currently marked with the status ”DEPRECATED” cannot accept new
evaluation metrics as it is no longer being monitored.
-Task:Create a ModelDeprecatedException . Map this to an HTTP 403 Forbidden status
when aPOSTmetric is attempted on a deprecated model.
4. The Global Safety Net (500) (5 Marks):
-Task:Implement a ”catch-all” ExceptionMapper <Throwable >. This mapper must intercept
anyunexpectedruntimeerrors(e.g., NullPointerException ,IndexOutOfBoundsException )

-Question: If an operation throws a specific custom exception
(e.g.,LinkedWorkspaceNotFoundException )andyoualsohaveaglobal ExceptionMapper <Throwable >,

-Question: Inyourfilter,youinteractwith ContainerRequestContext andContainerResponseContext .
